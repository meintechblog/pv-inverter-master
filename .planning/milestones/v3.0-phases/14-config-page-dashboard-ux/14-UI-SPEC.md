## Component Inventory

### New Components (Phase 14)

#### 1. Venus OS Config Section (`ve-panel`)
Location: Config page, below existing Inverter Configuration panel.

```
[Venus OS Configuration]
  Venus OS IP:     [_______________] (placeholder: "192.168.1.1")
  MQTT Port:       [_______________] (placeholder: "1883")
  Portal ID:       [_______________] (placeholder: "leave blank for auto-discovery")
```

- Uses existing `.ve-panel`, `.ve-form-group`, `.ve-input` classes
- Portal ID input gets `placeholder="leave blank for auto-discovery"` (italic style not needed, browser default placeholder is sufficient)
- Venus OS IP input: `type="text"`, `pattern` attribute for IP format validation
- MQTT Port input: `type="number"`, `min="1"`, `max="65535"`, default value `1883`

#### 2. Connection Bobbles (Config Page)
Location: Config page, inline with each section heading.

```
[*] SolarEdge Inverter    [Inverter Configuration]
[*] Venus OS MQTT         [Venus OS Configuration]
```

Bobble is a 12px `.ve-dot` placed inline with the panel `<h2>` heading, left of the text.

States:
- **Green + pulse**: connected (`ve-dot--ok` + `ve-pulse` animation)
- **Amber + pulse**: reconnecting/transient (`ve-dot--warn` + `ve-pulse` animation at 1.5s)
- **Red + static**: configured but disconnected (`ve-dot--err`)
- **Grey + static**: not configured (`ve-dot--dim`)

CSS addition for pulsing:
```css
@keyframes ve-pulse {
    0%, 100% { opacity: 1; }
    50% { opacity: 0.5; }
}
.ve-dot--ok { animation: ve-pulse 2s ease-in-out infinite; }
.ve-dot--warn { animation: ve-pulse 1.5s ease-in-out infinite; }
```

Data source: WebSocket snapshot `connection.state` (SolarEdge) and `venus_mqtt_connected` (Venus OS).

#### 3. MQTT Setup Guide Card (`ve-hint-card`)
Location: Config page, below Venus OS Configuration panel. Shown only when Venus host is configured but MQTT is disconnected.

Visual treatment:
- Background: `rgba(240, 150, 46, 0.12)` (orange tint, similar to existing `.ve-ctrl-revert`)
- Border: `1px solid rgba(240, 150, 46, 0.25)`
- Border-radius: `12px` (matches `.ve-card`)
- Padding: `16px`
- Icon: Inline SVG info icon (20x20) in `--ve-orange`, left of heading

Two states:
1. **Venus not configured** (host empty): Card hidden entirely.
2. **Venus configured but disconnected**: Card visible with step-by-step instructions.

#### 4. MQTT Gate Overlay (`.mqtt-gated`)
Location: Dashboard page. Applied to Venus-dependent widget containers.

Gated elements (Venus-dependent):
- `#venus-lock-container` (Venus OS Control toggle in gauge card)
- `#venus-ess-panel` (Venus OS ESS card)

NOT gated (inverter-only, must remain functional):
- Power gauge + phase cards + sparklines
- Inverter status panel
- Power control slider (min/max clamp dropdowns)
- Today's Performance
- Service Health
- Connection panel

CSS:
```css
.mqtt-gated {
    position: relative;
    opacity: 0.35;
    pointer-events: none;
    transition: opacity 300ms cubic-bezier(0.4, 0, 0.2, 1);
}
.mqtt-gated::after {
    content: 'Requires Venus OS MQTT';
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    color: var(--ve-text-dim);
    font-size: 0.85rem;
    text-align: center;
    white-space: nowrap;
    pointer-events: none;
}
```

The existing "Test Connection" button is removed. A single "Save & Apply" button saves both inverter and Venus config atomically. The button uses existing `.ve-btn--primary` styling.

## Interaction Contracts

### Save & Apply Flow
1. User fills inverter + Venus fields
2. User clicks "Save & Apply" (`.ve-btn--primary`)
3. Button shows "Saving..." (disabled state, text swap)
4. POST `/api/config` with `{ inverter: {...}, venus: {...} }`
5. On success: green toast "Configuration saved. Reconnecting..."
6. Button re-enables, text reverts to "Save & Apply"
7. Connection bobbles update from next WebSocket snapshot (~1s)

### Connection Bobble Update
- Driven by WebSocket snapshot data, not polling
- SolarEdge bobble reads `snapshot.connection.state` ("connected" | "reconnecting" | "disconnected")
- Venus OS bobble reads `snapshot.venus_mqtt_connected` (true | false) and config state (host empty = grey)
- Two consecutive matching snapshots required before bobble color change (debounce via CSS transition, not JS timer)

### MQTT Gate Toggle
- On each WebSocket snapshot, check `venus_mqtt_connected`
- If false: add `.mqtt-gated` to all `.venus-dependent` containers
- If true: remove `.mqtt-gated` from all `.venus-dependent` containers
- Transition is 300ms opacity fade (CSS handles it)

### Setup Guide Card Visibility
- Hidden when: Venus host is empty (not configured) OR Venus MQTT is connected
- Visible when: Venus host is non-empty AND Venus MQTT is disconnected
- Driven by same WebSocket snapshot + config form field value

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Reduced motion | Existing `@media (prefers-reduced-motion: reduce)` rule disables all animations including pulse |
| Focus visible | Existing `.ve-input:focus` ring (`box-shadow: 0 0 0 2px rgba(56,125,197,0.25)`) applies to new Venus inputs |
| MQTT gate | `pointer-events: none` prevents interaction. `aria-disabled="true"` added via JS to gated containers |
| Bobble meaning | Bobble color alone does not convey status. Adjacent text label ("SolarEdge: Connected" / "Venus OS: Disconnected") provides text equivalent |
| Form labels | All Venus inputs have associated `<label for="...">` elements |

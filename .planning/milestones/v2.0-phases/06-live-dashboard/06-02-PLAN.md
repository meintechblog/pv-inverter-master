**app.js changes:**

Replace the `setInterval` polling approach for dashboard page with WebSocket. Keep `pollRegisters()` as-is (only active when register page is shown). Keep config form as-is.

1. Add at top: `let ws = null; let sparklineData = []; const CAPACITY_W = 30000;`

2. Add `connectWebSocket()` function (see RESEARCH.md Pattern 4 for auto-reconnect):
   - Build URL: `const protocol = location.protocol === 'https:' ? 'wss:' : 'ws:'; new WebSocket(protocol + '//' + location.host + '/ws')`
   - `onopen`: reset reconnect delay, update ws-dot to green "WebSocket: Connected"
   - `onmessage`: parse JSON, route to `handleSnapshot(msg.data)` or `handleHistory(msg.data)` based on msg.type
   - `onclose`: update ws-dot to red "WebSocket: Disconnected", setTimeout reconnect with exponential backoff (1s, 2s, 4s...max 30s)
   - `onerror`: close ws (triggers onclose -> reconnect)

3. Add `handleSnapshot(data)`:
   - `const inv = data.inverter;`
   - Call `updateGauge(inv.ac_power_w || 0)`
   - Call `updatePhaseCard('l1', inv.ac_voltage_an_v, inv.ac_current_l1_a)`
   - Call `updatePhaseCard('l2', inv.ac_voltage_bn_v, inv.ac_current_l2_a)`
   - Call `updatePhaseCard('l3', inv.ac_voltage_cn_v, inv.ac_current_l3_a)`
   - Append to sparklineData, cap at 3600 entries, call `renderSparkline()`
   - Call `updateGaugeStatus(inv.status)`
   - Update connection/health from data.connection (replace pollStatus/pollHealth for dashboard page)
   - Update top-bar dots from data.connection.state

4. Add `handleHistory(data)`:
   - If data.ac_power_w exists: `sparklineData = data.ac_power_w.map(p => p[1])` (extract values only)
   - Call `renderSparkline()`

5. Add `updateGauge(powerW)`:
   - `const pct = Math.min(powerW / CAPACITY_W, 1.0)`
   - `const arcLength = 251.3` (semicircle path length for r=80)
   - `const offset = arcLength * (1 - pct)`
   - Set `gauge-fill` strokeDashoffset to offset
   - Set `gauge-value` text to `(powerW / 1000).toFixed(1)`
   - Color: green < 50%, orange 50-80%, red > 80% (use CSS vars)

6. Add `updatePhaseCard(phase, voltage, current)`:
   - Set `#{phase}-voltage` to `voltage.toFixed(1) + ' V'` (or '-- V' if null)
   - Set `#{phase}-current` to `current.toFixed(2) + ' A'` (or '-- A' if null)
   - Calculate power: `(voltage * current).toFixed(0) + ' W'` (or '-- W')
   - Flash animation: add `ve-value-flash` class, remove after 300ms

7. Add `renderSparkline()`:
   - Target: `#sparkline-power` SVG element
   - viewBox: 600 x 80
   - Generate polyline points from sparklineData (normalize to 0..80 y range with 10% padding)
   - Create/update SVG polyline (stroke: var(--ve-blue), width 1.5, fill none)
   - Create/update SVG polygon for fill area (same points + bottom corners, fill: var(--ve-blue), opacity 0.15)
   - Update sparkline-min and sparkline-max labels
   - Handle empty data gracefully

8. Add `updateGaugeStatus(status)`:
   - Set `#gauge-status` text to status string ("MPPT", "OFF", etc.)

9. Modify initialization in `DOMContentLoaded`:
   - Call `connectWebSocket()` instead of (or in addition to) polling
   - Keep `loadConfig()` call
   - Keep `pollRegisters()` but only run it when registers page is active (check if `#page-registers.active` exists)
   - Keep `pollStatus()` and `pollHealth()` as fallbacks but reduce to 10s interval (WebSocket provides live data for dashboard page; polling is fallback for status/health on non-dashboard pages)

10. Add `updateConnectionIndicator(state)` to update ws-dot:
    - 'connected': ve-dot--ok, text "WebSocket: Connected"
    - 'disconnected': ve-dot--err, text "WebSocket: Disconnected"

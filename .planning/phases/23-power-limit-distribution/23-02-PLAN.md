    **Add import** at top of __main__.py:
    ```python
    from venus_os_fronius_proxy.distributor import PowerLimitDistributor
    ```

    **In run_with_shutdown**, after creating DeviceRegistry but before starting devices:
    ```python
    # Create DeviceRegistry with aggregation callback
    registry = DeviceRegistry(app_ctx, config, on_poll_success=aggregation.recalculate)
    app_ctx.device_registry = registry

    # Create PowerLimitDistributor for multi-device power limiting
    distributor = PowerLimitDistributor(registry, config)
    ```

    **Problem:** run_modbus_server is called BEFORE DeviceRegistry is created (because it creates cache which AggregationLayer needs). The distributor needs the registry. Solution: create distributor after registry, then inject into slave_ctx post-hoc.

    Two options:
    A) Pass distributor to run_modbus_server (requires reordering -- risky)
    B) Set distributor on slave_ctx after creation (simpler, matches existing pattern)

    **Use option B:** run_modbus_server returns the slave_ctx (or we access it). Currently it returns `(cache, control_state, server, server_task)`. We need to either:
    - Return slave_ctx too, OR
    - Store slave_ctx on app_ctx, OR
    - Access it via server.context

    Simplest: modify run_modbus_server to also return slave_ctx:
    ```python
    return cache, control_state, server, server_task, slave_ctx
    ```

    Then in __main__.py:
    ```python
    cache, control_state, server, server_task, slave_ctx = await run_modbus_server(
        host=config.proxy.host,
        port=config.proxy.port,
        app_ctx=app_ctx,
    )
    aggregation = AggregationLayer(app_ctx, cache, config)
    registry = DeviceRegistry(app_ctx, config, on_poll_success=aggregation.recalculate)
    app_ctx.device_registry = registry
    distributor = PowerLimitDistributor(registry, config)
    slave_ctx._distributor = distributor
    ```

    This avoids changing run_modbus_server's parameter signature (keeps it backward-compatible) and just sets the distributor after the fact.

    **Log the distributor creation:**

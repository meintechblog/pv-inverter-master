**src/venus_os_fronius_proxy/plugins/opendtu.py (write SECOND — make tests pass):**

```python
"""OpenDTU plugin for Hoymiles micro-inverters.

Polls Hoymiles inverters via OpenDTU REST API and translates JSON responses
to SunSpec uint16 register arrays for DashboardCollector compatibility.
Supports power limiting via POST /api/limit/config with dead-time guard.
"""
from __future__ import annotations

import time
from dataclasses import dataclass

import aiohttp
import structlog

from venus_os_fronius_proxy.config import GatewayConfig
from venus_os_fronius_proxy.plugin import InverterPlugin, PollResult, WriteResult
from venus_os_fronius_proxy.sunspec_models import (
    encode_string,
    _int16_as_uint16,
    COMMON_DID,
    COMMON_LENGTH,
    NAMEPLATE_DID,
    NAMEPLATE_LENGTH,
)
```

Constructor takes: `gateway_config: GatewayConfig, serial: str, name: str = ""`

Key implementation details:

1. **connect()**: Create `aiohttp.ClientSession` with `aiohttp.BasicAuth(gateway_config.user, gateway_config.password)`. Store as `self._session`. Optionally fetch `/api/limit/status` to get `max_power` for Model 120 WRtg.

2. **poll()**:
   - `GET http://{gateway_host}/api/livedata/status`
   - Parse JSON, find inverter by serial in `data["inverters"]`
   - If not found or `reachable == False`: return `PollResult([], [], success=False, error=...)`
   - Extract AC values from `inv["AC"]["0"]` using defensive `.get()` chains
   - Extract DC values: sum Power across all DC channels (0, 1, ...), sum Current, use power-weighted average for Voltage
   - Extract YieldTotal: sum across DC channels, convert from kWh to Wh (* 1000, round to int)
   - Extract YieldDay: sum across DC channels, convert from kWh to Wh
   - Extract Temperature from `inv["INV"]["0"]["Temperature"]["v"]`
   - Status: 4 (MPPT) if producing, 2 (SLEEPING) if not producing but reachable
   - Call `_encode_model_103(...)` to build 52 uint16 registers
   - Call `_build_common_registers()` to build 67 uint16 registers
   - Return `PollResult(common_regs, inverter_regs, success=True)`

3. **_encode_model_103(ac_power_w, ac_voltage_v, ac_current_a, ac_freq_hz, dc_power_w, dc_voltage_v, dc_current_a, temperature_c, energy_total_wh, yield_day_wh, status_code) -> list[int]:**
   - Create 52-element list of zeros
   - regs[0] = 103 (DID), regs[1] = 50 (Length)
   - Use scale factors from 21-RESEARCH.md register offset table:
     - AC Current (offset 2): `int(round(ac_current_a * 100))`, SF at offset 6: `_int16_as_uint16(-2)`
     - AC Current L1 (offset 3): same as total (single-phase micro-inverter)
     - AC Voltage AN (offset 10): `int(round(ac_voltage_v * 10))`, SF at offset 13: `_int16_as_uint16(-1)`
     - AC Power (offset 14): `int(round(ac_power_w))`, SF at offset 15: `0`
     - AC Frequency (offset 16): `int(round(ac_freq_hz * 100))`, SF at offset 17: `_int16_as_uint16(-2)`
     - AC Energy (offset 24-25): acc32 in Wh, high word at 24, low word at 25, SF at offset 26: `0`
     - DC Current (offset 27): `int(round(dc_current_a * 100))`, SF at offset 28: `_int16_as_uint16(-2)`
     - DC Voltage (offset 29): `int(round(dc_voltage_v * 10))`, SF at offset 30: `_int16_as_uint16(-1)`
     - DC Power (offset 31): `int(round(dc_power_w))`, SF at offset 32: `0`
     - Temperature Cab (offset 33): `int(round(temperature_c * 10))`, SF at offset 37: `_int16_as_uint16(-1)`
     - Status (offset 38): status_code

4. **_build_common_registers() -> list[int]:**
   - 67-element list: [COMMON_DID, COMMON_LENGTH, ...65 data registers]
   - Manufacturer: "Hoymiles" encoded into 16 registers starting at offset 2
   - Model: self.name or self.serial encoded into 16 registers at offset 18
   - Serial: self.serial encoded into 16 registers at offset 50

5. **get_static_common_overrides() -> dict[int, int]:**
   - Return overrides for Common Model registers (offset -> value)
   - Manufacturer string "Hoymiles", Model from name/serial

6. **get_model_120_registers() -> list[int]:**
   - 28-element list: [NAMEPLATE_DID, NAMEPLATE_LENGTH, ...26 data registers]
   - WRtg (offset 4): `self._max_power_w` (from /api/limit/status or default 400)
   - WRtg_SF (offset 5): 0

7. **write_power_limit(enable, limit_pct) -> WriteResult:**
   - Dead-time check: if `self._limit_pending` and `time.monotonic() - self._last_limit_ts < DEAD_TIME_S` (30.0), return `WriteResult(success=True)` with log message "dead_time_suppressed"
   - If not enable: set limit_pct to 100.0 (remove limit)
   - POST to `http://{gateway_host}/api/limit/config` with JSON body: `{"serial": self.serial, "limit_type": 1, "limit_value": limit_pct}`
   - Auth: already on session via BasicAuth
   - On success: set `self._last_limit_ts = time.monotonic()`, `self._limit_pending = True`
   - Return WriteResult

8. **reconfigure(host, port, unit_id):**
   - For OpenDTU, host/port/unit_id are not used (config is gateway_host + serial)
   - Close existing session, allow reconnect on next poll
   - Log warning that reconfigure is a no-op for OpenDTU (host/port/unit_id are SolarEdge concepts)

9. **close():**
   - Close `self._session` if not None and not closed
   - Log "opendtu_plugin_closed"

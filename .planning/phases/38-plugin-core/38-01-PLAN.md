    `def _parse_sungrow_data(self, raw: list[int]) -> dict` -- Parse 36 registers at wire 5002-5037. Offsets relative to raw[0]=wire 5002:

    - dc1_voltage_v: `raw[8] * 0.1` (wire 5010)
    - dc1_current_a: `raw[9] * 0.1` (wire 5011)
    - dc2_voltage_v: `raw[10] * 0.1` (wire 5012)
    - dc2_current_a: `raw[11] * 0.1` (wire 5013)
    - total_dc_power_w: `(raw[14] << 16) | raw[15]` (U32 wire 5016-5017)
    - phase_a_voltage_v: `raw[16] * 0.1` (wire 5018)
    - phase_b_voltage_v: `raw[17] * 0.1` (wire 5019)
    - phase_c_voltage_v: `raw[18] * 0.1` (wire 5020)
    - phase_a_current_a: `raw[19] * 0.1` (wire 5021)
    - phase_b_current_a: `raw[20] * 0.1` (wire 5022)
    - phase_c_current_a: `raw[21] * 0.1` (wire 5023)
    - total_active_power_w: `(raw[28] << 16) | raw[29]` (U32 wire 5030-5031)
    - power_factor: `_s16(raw[32]) * 0.001` (wire 5034)
    - frequency_hz: `raw[33] * 0.1` (wire 5035)
    - running_state: `raw[35]` (wire 5037)

    Helper `_s16(val: int) -> int` at module level: `return val if val < 0x8000 else val - 0x10000`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Proxy -> Sungrow Inverter | Modbus TCP on local LAN (192.168.x.x), no authentication in Modbus protocol |
| Poll data -> SunSpec encoding | Untrusted register values from device translated to uint16 array |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-01 | Spoofing | Modbus TCP connection | accept | Local LAN only, no internet exposure. Modbus TCP has no auth by design. Same risk as SolarEdge plugin (accepted in v1.0). |
| T-38-02 | Tampering | Raw register values | mitigate | Clamp all parsed values to sane ranges (voltage 0-1000V, current 0-200A, power 0-50kW, temp -40 to 100C). Use abs() for power/current per PLUGIN-CHECKLIST. |
| T-38-03 | Denial of Service | Modbus TCP timeout | mitigate | pymodbus has built-in timeout (3s default). poll() wraps in try/except, returns error PollResult. ConnectionManager handles reconnect backoff. |
| T-38-04 | Information Disclosure | Register data in logs | accept | Only structlog info-level logs connection events. No raw register dumps in production log level. |
| T-38-05 | Elevation of Privilege | write_power_limit no-op | accept | Phase 38 is read-only. write_power_limit returns success without writing. No privilege escalation possible. |
</threat_model>

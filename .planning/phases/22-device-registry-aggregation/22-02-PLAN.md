Create the AggregationLayer that sums all active inverter outputs into one virtual Fronius inverter for Venus OS, and rewire proxy.py + __main__.py to use DeviceRegistry + AggregationLayer instead of the single-plugin architecture.

Output: `aggregation.py` with AggregationLayer, updated `proxy.py`/`__main__.py`/`config.py`/`context.py`/`webapp.py`, comprehensive tests.

Keys: ac_current_a, ac_current_l1_a, ac_current_l2_a, ac_current_l3_a, ac_voltage_an_v, ac_power_w, ac_frequency_hz, energy_total_wh, dc_current_a, dc_voltage_v, dc_power_w, temperature_c, status_code.

2. **encode_aggregated_model_103(totals: dict) -> list[int]** -- encodes aggregated physical values to 52 uint16 registers with FIXED scale factors: SF=0 power, SF=-2 current, SF=-1 voltage, SF=-2 frequency, SF=0 energy, SF=-1 temperature. 3. **AggregationLayer class**:
   - `__init__(self, app_ctx: AppContext, cache: RegisterCache, config: Config)` -- stores refs
   - `async recalculate(self, device_id: str) -> None` -- the on_poll_success callback. Called after each successful device poll.
     - Take snapshot: `device_ids = list(self._app_ctx.devices.keys())`
     - Collect active states: those where ds.last_poll_data is not None and has "inverter_registers" key
     - If no active states: return (cache stays stale, no update)
     - For each active state: decode inverter_registers using decode_model_103_to_physical
     - Sum: ac_power_w, ac_current_a, ac_current_l1/l2/l3_a, energy_total_wh, dc_current_a, dc_power_w

     - Max: temperature_c
     - Status: use worst (highest) status_code across devices (4=MPPT is best, higher is worse)
     - Encode using encode_aggregated_model_103
     - Build virtual Common Model registers: Manufacturer="Fronius", Model=config.virtual_inverter.name (or default), use encode_string from sunspec_models
     - cache.update(COMMON_CACHE_ADDR, virtual_common)
     - cache.update(INVERTER_CACHE_ADDR, aggregated_inverter)
     - Update Model 120 WRtg: sum rated_power from config.inverters where enabled=True and id is in active device_ids. Write to datablock at 40124+1 offset.
   - `update_wrtg(self)` -- recalculate and write WRtg to datablock. Called after device add/remove.

Replace single-plugin architecture with DeviceRegistry + AggregationLayer:

```python
async def run_with_shutdown():
    # ... signal handlers ...

    # Create Modbus server infrastructure (no plugin needed)
    cache, control_state, server, server_task = await setup_modbus_server(
        config, app_ctx
    )

    # Create AggregationLayer
    aggregation = AggregationLayer(app_ctx, cache, config)

    # Create DeviceRegistry with aggregation callback
    registry = DeviceRegistry(app_ctx, config, on_poll_success=aggregation.recalculate)
    app_ctx.device_registry = registry

    # Start all enabled devices
    await registry.start_all()

    # If no active devices, stop Modbus server
    if registry.get_active_count() == 0:
        # Stop server -- Venus OS won't find us
        server_task.cancel()
        ...

    # Start webapp (pass registry instead of single plugin)
    ...

    # Shutdown: registry.stop_all(), server cleanup, webapp cleanup
```

The Modbus server start/stop logic based on active device count:
- After start_all(): if 0 active, do not start server (or cancel server task)
- After device add/enable: if server not running and active_count > 0, start server
- After device remove/disable: if active_count == 0, stop server

For simplicity in this phase, implement server stop at startup when 0 devices. Dynamic start/stop during runtime can use a helper method on a new orchestration object or directly in the webapp handlers.

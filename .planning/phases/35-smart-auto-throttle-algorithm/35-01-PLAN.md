<objective>
Implement the core smart auto-throttle algorithm: score-based waterfall ordering, convergence tracking, and effective score override.

Purpose: When auto_throttle is enabled, the distributor automatically selects the optimal throttle order -- fastest proportional devices regulate first, binary devices are last resort. Live convergence measurement lets scores adapt to real-world device behavior.

Output: Modified config.py (auto_throttle field), distributor.py (score-based waterfall + convergence tracking), and comprehensive tests.
</objective>

       ```python
       def _effective_score(self, ds: DeviceLimitState) -> float:
           if not hasattr(ds.plugin, 'throttle_capabilities'):
               return 0.0
           caps = ds.plugin.throttle_capabilities
           if ds.measured_response_time_s is not None:
               measured_caps = ThrottleCaps(
                   mode=caps.mode,
                   response_time_s=ds.measured_response_time_s,
                   cooldown_s=caps.cooldown_s,
                   startup_delay_s=caps.startup_delay_s,
               )
               return compute_throttle_score(measured_caps)
           return compute_throttle_score(caps)
       ```
